# Security Policy (Word-to-PDF Project)

## Supported Versions

Security fixes are applied to the latest mainline state of the repository.

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Report privately with:
- A clear description of the vulnerability
- Reproduction steps or proof-of-concept
- Impact assessment
- Suggested mitigation (if available)

If a private channel is not configured yet, open a minimal issue asking maintainers to provide a secure contact path, without disclosing details.

## Response Expectations

- Initial acknowledgment target: within 7 days
- Triage and impact assessment: as soon as feasible
- Fix timeline: depends on severity and complexity

## Disclosure

After a fix is available, maintainers may publish a coordinated disclosure note including affected versions and upgrade guidance.
